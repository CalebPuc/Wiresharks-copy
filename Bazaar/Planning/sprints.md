Sprint #1: Begin by constructing the Bazaar game rules. Start with a chip class with basic attributes and methods. Develop a class for the cards, then develop child classes for the target cards and equation cards. Develop a class for the player and define their actions as methods with respect to each card. Develop a class for the referee and define their actions as methods. Develop the observers class to allow bazaar.com to sell ads to those observers. Develop the bank class and add methods to allow it to keep track of players’ inventory. Finally, develop a player-referee interface to help start the game.

Sprint #2: Begin setting up the interface between potential players and the game. Construct the API needed to allow proper communication between players and the game. Make sure this API fits the requirements of a plug-in protocol and a sign-up period. Define each potential player beforehand with a new class to differentiate them from the other players. 

Sprint #3: Polish the code, looking back at everything to ensure all classes are in good shape. Develop the graphical interface for the bazaar game to make it more appealing to users (if the time spares and all potential bugs have been spotted).
